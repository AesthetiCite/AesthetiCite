vision_engine_v2.py → app/api/vision_engine_v2.py — improvements 1, 2, 3, 5, 7, 8, 9, 10
_get_vision_model() routes to Claude 3.7 Sonnet or GPT-4.5 based on VISION_MODEL env var — a single env change swaps the entire vision backend. preprocess_image() runs PIL auto-contrast, unsharp mask, and colour enhancement before every analysis call. _stream_vision_analysis() emits SSE events as tokens arrive — status phases, live content, visual scores, and protocol alerts all surface mid-stream. The multi-image endpoint sends before + after to the model simultaneously in one call. Video analysis extracts evenly-spaced frames via OpenCV, runs the signal detector on each, and returns a trajectory narrative. DICOM export wraps the analysed image as a clinical-standard DCM file using pydicom. CLIP embeddings power the similarity search — cosine distance against the embedding store returns the most clinically similar historical cases. The calibration endpoint computes per-field correction offsets from clinician-labelled cases and applies them via apply_calibration().
vision_landmarks.py → app/api/vision_landmarks.py — improvement 4
Downloads the OpenCV Haar cascade once on first use (no model download required, no dlib). _detect_face_and_zones() divides the detected face into 9 anatomical zones: forehead, glabella, bilateral periorbital, nose, bilateral cheeks, lips/perioral, chin/jawline. Each zone is cropped and sent to GPT-4o with a zone-specific prompt that names the relevant danger zones and expected complications for that region. Returns localised signals — "blanching in glabella zone near supratrochlear artery" instead of "blanching visible." The /landmark-preview endpoint returns an annotated JPEG with zone outlines drawn, useful for debugging and clinical demonstration.
vision-sw.js → public/vision-sw.js — improvement 6 + streaming UI
The service worker caches the Vision capture pages for offline use, queues analysis requests in IndexedDB when offline, and replays them automatically when connectivity is restored. Safety-critical routes (protocols, safety check, ask) always go network-first and are never cached or queued. The VisionStreamingAnalysis.tsx component is embedded as a commented block at the bottom of the file — extract it to client/src/components/vision/VisionStreamingAnalysis.tsx. It consumes the SSE stream from /v2/stream, renders tokens in real time, surfaces protocol alerts mid-analysis before the full text is complete, and shows a queued/offline state when the service worker intercepts the request.
