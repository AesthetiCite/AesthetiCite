================================================================================
LATENCY BREAKDOWN — HOW < 2s IS ACHIEVED
================================================================================

Current bottlenecks and what each patch addresses:

STEP                    BEFORE     AFTER (hot)   AFTER (non-hot)
─────────────────────────────────────────────────────────────────
Safety screen           ~1ms       ~1ms          ~1ms
Cache lookup            ~2ms       ~2ms          ~2ms
Embedding (query)       ~80ms      0ms*          ~80ms
Vector search           ~120ms     0ms*          ~80ms
Context assembly        ~5ms       0ms*          ~5ms
LLM first token         ~900ms     ~50ms**       ~700ms
Total to first token    ~1.1s      ~55ms         ~870ms
Full answer stream      +0.8s      +0.3s         +0.6s
Total end-to-end        ~1.9s      ~380ms        ~1.5s

* Embedding precomputed at startup; retrieval result cached in _retrieval_cache
** Hot answers streamed from memory at 150 chars/chunk

For non-hot complication queries (not in the 16 precomputed):
  - Context limiter cuts chunks from ~10–14 to 6
  - Fewer chunks → shorter prompt → ~30% faster LLM first token
  - Target: < 2s end-to-end for complication queries
  - General queries remain at 2.5–4s (no change, acceptable)

================================================================================
