PATCH 6 — client/src/pages/ask.tsx  (main evidence search page)
================================================================================

ADD import:

    import { SafetyEscalationBlock, useSafety } from "@/components/safety-escalation-block";

FIND where `aciMeta` is defined (the SSE meta payload object).

ADD after the answer rendering (at the same level as ComplicationProtocolAlert):

    {/* Safety block — always shown, adapts to risk level of the query */}
    {(() => {
      const safety = useSafety(aciMeta);  // ← hook reads meta.safety
      return safety ? (
        <SafetyEscalationBlock
          safety={safety}
          className="my-4"
        />
      ) : null;
    })()}

NOTE: useSafety() is a simple accessor function, not a real React hook
(doesn't use useState/useEffect). You can call it inline like above,
or just access the data directly:

    {aciMeta?.safety && (
      <SafetyEscalationBlock safety={aciMeta.safety} className="my-4" />
    )}

================================================================================
WHAT CHANGES — BEFORE vs AFTER
================================================================================

BEFORE:
  - Safety only fires for complication queries
  - escalation_note is a plain string appended to the prompt
  - Frontend shows escalation buried in the answer text
  - No structured `when_to_stop` or `when_to_refer` fields
  - Different format per endpoint (some have it, some don't)
  - No medico-legal documentation checklist
  - No risk level colour coding

AFTER:
  - Safety fires on EVERY query (complication, anatomy, technique, general)
  - safety_block_for_prompt() injects structured rules into every LLM prompt
  - enrich_meta_payload() adds safety to every SSE meta event
  - Frontend renders SafetyEscalationBlock after every answer
  - Risk level drives visual emphasis (critical = red expanded, low = grey collapsed)
  - Structured when_to_stop / when_to_refer / escalation_criteria per query type
  - Medico-legal checklist always included (collapsible)
  - Zero added latency (all rule-based, < 1ms)
