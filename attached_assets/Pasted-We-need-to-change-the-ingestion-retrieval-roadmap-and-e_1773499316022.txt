We need to change the ingestion/retrieval roadmap and execution order for the AesthetiCite corpus engine.

Context
There is one ingestion engine with 4 sequential phases:

M1 = quality cleanup on existing corpus
M2 = guideline-first ingestion
M3 = multilingual ingestion
M4 = full breadth sweep

Current state

- M1 exists and runs quickly
- M3 is currently running
- M2 was skipped because corpus size already exceeded the 250k target
- Audit findings show:
  1. multilingual retrieval is broken because French/Spanish are being treated like native retrieval against a 99.8% English corpus
  2. retrieval is slow
  3. ACI is low because guideline docs are not being surfaced strongly enough
  4. 8,618 docs are mislabeled
  5. citation URLs are not being populated correctly
