5) FRONTEND — Register the service worker
-----------------------------------------
ADD this to:
    client/src/main.tsx

AFTER app mount:

if ("serviceWorker" in navigator) {
  window.addEventListener("load", () => {
    navigator.serviceWorker
      .register("/sw.js", { scope: "/" })
      .then((reg) => console.log("[PWA] Service worker registered", reg.scope))
      .catch((err) => console.warn("[PWA] Service worker registration failed", err));
  });
}

WHY:
- without this, the production service worker will not activate

6) FRONTEND — Fix clinic query logging
--------------------------------------
IN:
    client/src/pages/ask.tsx

FIND the old function:
    logQueryToClinic(question, answerPreview, aci, durationMs)

REPLACE with:

function logQueryToClinic(question: string, aci: number | null, durationMs: number) {
  const token = getToken();
  if (!token) return;

  fetch("/api/ops/dashboard/log-query", {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      Authorization: `Bearer ${token}`,
    },
    body: JSON.stringify({
      query_text: question,
      answer_type: "evidence_search",
      aci_score: aci ?? undefined,
      response_time_ms: Math.round(durationMs),
      domain: "aesthetic_medicine",
    }),
  }).catch(() => {});
}

UPDATE the call sites to:
    logQueryToClinic(question, aciScore, Date.now() - queryStartTimeRef.current);

WHY:
- clinic logging must derive clinic identity server-side from authenticated user
- do not trust localStorage clinic_id
