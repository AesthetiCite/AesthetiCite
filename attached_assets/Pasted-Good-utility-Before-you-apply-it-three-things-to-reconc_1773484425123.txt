Good utility. Before you apply it, three things to reconcile with your specific routes.ts structure:

Issue 1 — Your streams use typed SSE events, not a single final payload
The proxyStreamSafe function intercepts payload.answer || payload.response || payload.text — a flat JSON pattern. But your v2 stream emits typed event objects:
json{ "type": "content", "data": "token..." }
{ "type": "citations", "citations": [...] }
{ "type": "done" }
The sanitizer as written will pass the citations event through unchecked because it has no answer field. You need to intercept on type === "citations" instead:
typescript// Replace the inner condition with this
if (payload.type === "citations" || payload.type === "meta") {
  const safe = sanitizeGroundedResponse(payload);
  res.write(`data: ${JSON.stringify(safe)}\n\n`);
} else if (payload.answer || payload.response || payload.text) {
  // flat JSON fallback (covers /api/ask stream)
  const safe = sanitizeGroundedResponse(payload);
  res.write(`data: ${JSON.stringify(safe)}\n\n`);
} else {
  res.write(`data: ${JSON.stringify(payload)}\n\n`);
}
