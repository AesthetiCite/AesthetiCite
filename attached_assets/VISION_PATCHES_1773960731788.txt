PATCH 2 — server/routes.ts
================================================================================

ADD inside registerRoutes() — these are multipart form routes (like existing visual upload):

    // ── AesthetiCite Vision — VisualDX-style diagnosis ───────────────────
    app.post("/api/vision/diagnose", upload.single("file"), async (req, res) => {
      try {
        const formData = new FormData();
        if (req.file) {
          formData.append("file", req.file.buffer, {
            filename: req.file.originalname || "photo.jpg",
            contentType: req.file.mimetype,
          });
        }
        // Pass clinical context fields
        ["procedure", "region", "time_since", "product"].forEach((key) => {
          if (req.body[key]) formData.append(key, req.body[key]);
        });
        const response = await fetch(`${PYTHON_API_BASE}/api/vision/diagnose`, {
          method: "POST",
          headers: {
            ...formData.getHeaders(),
            ...(req.headers.authorization ? { Authorization: req.headers.authorization as string } : {}),
          },
          body: formData as unknown as BodyInit,
        });
        const data = await response.json();
        res.status(response.status).json(data);
      } catch (error) {
        res.status(502).json({ detail: "Vision diagnosis unavailable" });
      }
    });

    app.post("/api/vision/diagnose-compare", upload.fields([
      { name: "baseline", maxCount: 1 },
      { name: "followup", maxCount: 1 },
    ]), async (req, res) => {
      try {
        const files = req.files as Record<string, Express.Multer.File[]>;
        const formData = new FormData();
        const baseFile = files["baseline"]?.[0];
        const followFile = files["followup"]?.[0];
        if (!baseFile || !followFile) {
          return res.status(400).json({ detail: "Both baseline and followup images required." });
        }
        formData.append("baseline", baseFile.buffer, {
          filename: baseFile.originalname || "baseline.jpg",
          contentType: baseFile.mimetype,
        });
        formData.append("followup", followFile.buffer, {
          filename: followFile.originalname || "followup.jpg",
          contentType: followFile.mimetype,
        });
        ["procedure", "region", "days_since_baseline"].forEach((key) => {
          if (req.body[key]) formData.append(key, req.body[key]);
        });
        const response = await fetch(`${PYTHON_API_BASE}/api/vision/diagnose-compare`, {
          method: "POST",
          headers: {
            ...formData.getHeaders(),
            ...(req.headers.authorization ? { Authorization: req.headers.authorization as string } : {}),
          },
          body: formData as unknown as BodyInit,
        });
        const data = await response.json();
        res.status(response.status).json(data);
      } catch (error) {
        res.status(502).json({ detail: "Vision compare unavailable" });
      }
    });

    app.get("/api/vision/complication-signs", (req, res) =>
      proxyToFastAPI(req, res, "/api/vision/complication-signs")
    );

HOW THIS DIFFERS FROM EXISTING VISUAL FEATURES
================================================================================

                    Visual Counseling    Vision Follow-up    AesthetiCite Vision
                    (/visual-counsel)    (/vision-followup)  (NEW)
─────────────────────────────────────────────────────────────────────────────
Input               1 photo (pre-proc)   2 photos (serial)   1 OR 2 photos
Output              Free-text evidence   Pixel diff metrics  Ranked differential
Format              Streaming SSE text   Numeric scores      Structured JSON
Complication named? No (in answer text)  No                  YES — named + ranked
Urgency classified? No                   No                  YES — immediate/urgent
Protocol bridge?    No                   No                  YES → /decide?complication=
Visual evidence?    No                   No                  YES — what AI observed
Differential list?  No                   No                  YES — 4 alternatives
Action given?       Generic evidence     None                Specific next step
─────────────────────────────────────────────────────────────────────────────

The three features are COMPLEMENTARY, not competing:
  Visual Counseling  → evidence-backed consultation report (pre-procedure)
  Vision Follow-up   → objective measurement over time (post-procedure)
  AesthetiCite Vision → VisualDX-style named complication (any stage)

AesthetiCite Vision sits inside both existing pages as an additional layer.


================================================================================
THE COMPLICATION SIGN DICTIONARY — WHAT MAKES IT VisualDX-LIKE
================================================================================

COMPLICATION_SIGNS in vision_diagnosis.py is the proprietary clinical asset:
  9 complication types
  Each with: display name, urgency, risk level, 5 specific visual signs,
             clinical action, trigger protocol key

This dictionary:
  1. Grounds the LLM prompt (tells GPT-4o exactly what to look for)
  2. Enriches the LLM output (adds action text + protocol key to result)
  3. Is served via GET /api/vision/complication-signs for the frontend "what to look for" guide
  4. Grows with every case logged → future fine-tuning dataset

This is what VisualDX built over 20 years (their library is 30,000+ conditions).
AesthetiCite Vision starts with 9 aesthetic-specific conditions — the exact ones that matter.
