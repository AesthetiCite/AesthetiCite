GOAL

support@aestheticite.com

STACK
Use the existing stack already in this project:
- Express server
- React client

ACCESS CONTROL

BACKEND
Create admin endpoints under /api/admin with proper auth protection.

Required endpoints:
1. GET /api/admin/overview
Return:
- totalUsers
- activeUsers24h
- activeUsers7d
- totalQueries
- totalAnswers
- totalCaseLogs
- totalSafetyReports
- avgQueriesPerUser
- avgResponseTimeMs if available, otherwise null
- topComplications: array of { name, count }
- topProtocols: array of { name, count }
- recentActivity: latest 20 events mixing signups, queries, case logs, safety reports when available

2. GET /api/admin/users
Return a paginated list of users with:
- id
- email
- name if available
- createdAt
- lastSeenAt if available
- queryCount
- answerCount
- caseLogCount
- role = "admin" for support@aestheticite.com else "user"

3. GET /api/admin/analytics
Return:
- dailyQueriesLast30d
- dailyActiveUsersLast30d
- complicationDistribution
- protocolDistribution
- evidenceUsage if available
- queryLanguageDistribution if available

4. GET /api/admin/recent-logins
If explicit login tracking does not exist, infer from most recent authenticated activity and return:
- email
- lastActivityAt
- source = inferred or actual

5. GET /api/admin/health
Return:
- dbConnected
- totalDocuments
- totalChunks
- chunkedDocuments
- documentsWithoutChunks
- latestQueryAt
- latestCaseLogAt

DATABASE / DATA MAPPING
Use existing tables whenever present:
- users
- queries
- answers
- case_logs
- safety_reports
- analytics_events
- saved_protocols
- documents
- chunks

If a field does not exist, infer it safely or return null.
Do not invent fake data.
Do not fail if some tables are sparse.
Use efficient SQL / Drizzle queries and avoid N+1 patterns.

IMPORTANT DATA RULES
- activeUsers24h / 7d = users with any recent activity in queries, answers, analytics_events, case_logs, or safety_reports
- topComplications should come primarily from case_logs.complication_type or nearest equivalent column
- topProtocols should come from saved_protocols or protocol usage in logs if available
- recentActivity should normalize records into:
  { type, label, email, createdAt, metadata }
- documentsWithoutChunks = documents left joined to chunks where no chunk exists
- avgQueriesPerUser = totalQueries / totalUsers rounded to 2 decimals
